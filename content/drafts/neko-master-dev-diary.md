上个月，我把 Clash Master 改名为 **Neko Master** 重新发布。项目上线一周收获了 1400+ GitHub Star，Docker Pull 数也破了 10K。趁 v1.2.7 版本刚发布（支持 Surge 了），写篇文章聊聊这个项目背后的故事。

## 为什么会有这个项目?

我是个 Homelab 用户，家里跑了一堆服务。

日常开发需要频繁切换 IP，家里的分流策略也比较复杂。原生的 Clash 面板更多是"运行状态"展示，但我缺少一个更直观的视角去看：

**流量到底在干什么？**

- 哪些域名在吃带宽？
- 哪个节点负载更高？
- 当前的规则策略是否合理？

与其在不同工具之间拼凑数据，不如自己做一个更聚焦"流量感知"的面板。

## 它是什么?

Neko Master 是一个开源、自部署的网络流量分析面板：

- **多维流量统计**: 域名 / 节点 / 规则 / ASN / 地理位置
- **趋势分析**: 30 分钟到 30 天的历史趋势
- **多后端支持**: OpenClash / Mihomo / Surge
- **旁路部署**: 非侵入式设计，不影响原网关稳定性
- **Docker 一键部署**: 5 分钟跑起来
- **移动端适配 + PWA**

定位很简单：**轻量、聚合分析优先、界面尽量现代。**

## Vibe Coding 实践

这个项目算是一次完整的 **Vibe Coding** 实践。

MVP 花了 **1 小时**完成，**4 小时**上线第一版。后续两周主要在做结构重构和稳定性打磨。

开发过程中用到了：

- **Kimi K2.5**: 早期结构搭建主力，中文理解好
- **Claude Code Opus 4.5 / 4.6**: 复杂逻辑与性能问题
- **GPT 5.2 / 5.3**: 迭代优化
- **Gemini 3 Pro**: 辅助 Review

一个真实体感是：**AI 非常适合快速把想法变成可运行的系统。**

国产模型在初始化阶段已经足够高效，但当复杂度提升时，Opus 4.6 / GPT 5.3 在 Debug 和架构层面的稳定性优势更明显。

### 关于"去 AI 味"

很多人吐槽 AI 生成的界面"千篇一律"。

我的小技巧是：**不要只对 AI 说"写个好看的面板"，给它具体的视觉参考。**

比如我会直接甩几张截图："参考这个配色"、"这个卡片阴影效果"、"这个数据可视化的风格"。设计质量会明显提升。

## 技术踩坑记录

项目虽小，但踩的坑不少。

### SQLite 写入优化

第一版的 SQLite 写入逻辑有问题，大量 I/O 导致一天写入 **200GB+**。用户反馈后紧急优化：

```
v1.0.4 → v1.0.7 → v1.0.9
  ↓       ↓        ↓
200GB   20GB    1.6GB
```

引入了批量写入 + 限流 + 内存缓存，I/O 降低了 **99%**。

### WebSocket 实时推送

早期版本用 HTTP 轮询（5 秒一次），延迟明显。后来加了 WebSocket 支持，实时性大幅提升。

但 WebSocket 的配置对用户来说是个门槛，需要反向代理支持。最后还是保留了 HTTP 轮询作为 fallback。

### Docker 镜像体积

早期镜像 **800MB+**，被用户吐槽。后来用 `pnpm deploy` 精简，现在降到 **300MB+**。

## 关于 Logo

Logo 意外收到不少好评，甚至有人专门在 V2EX 发帖讨论是怎么做的。

第一版用 Gemini 生成，后续 GPT 多轮优化细节。全程没有使用传统设计工具。

一个感受是：**在 AI 时代，代码的门槛在下降，但审美判断依然是决定成品质量的关键。**

## 部署建议

很多同学问我部署在哪。

**强烈建议旁路部署**，不要和网关（OpenWrt/软路由）放在同一台机器上。

这个项目有持续的 I/O 读写，虽然优化后已经大幅降低，但万一出问题影响主路由，家里网络就仆街了。

我自己是部署在 NAS（N5105 CPU）上，监控 3 个上游 Clash 网关，目前运行稳定。

## 一些数据

上线两周的数据：

- **GitHub Star**: 1400+
- **Docker Pull**: 10K+
- **Issues**: 大部分为配置问题
- **代码行数**: 约 1.5 万行（含注释）

一个有趣的观察：大部分用户其实不是开发者，而是普通的 Homelab 爱好者。他们更关心「能不能一键跑起来」、「占多少资源」、「会不会影响主路由」。

这也让我调整了项目的优先级：**易用性 > 功能丰富度**。

## 写在最后

这个项目从想法到上线，再到收到用户的反馈和 Star，整个过程挺有意思的。

AI 确实改变了开发方式。过去需要一周的原型，现在几小时就能跑起来。但**从"能跑"到"好用"，依然需要大量的打磨和思考**。
